Clinical Shift Scratchpad

A small, local-only mobile prototype for keeping transient clinical shift jobs out of scraps of paper, WhatsApp-to-self, screenshots, and overloaded memory.

View source

What it is

A structured scratchpad for active jobs: quick free-text entry, optional minimum-necessary patient identifier, location, tags, urgency, status, pinning, duplication, and local expiry.

Why it exists

Ward work often degrades into fragments. This tries to make the temporary list faster to capture, easier to scan, and less likely to drift during a shift.

Current status

Prototype, field-tested personally, and still deliberately narrow. Useful enough to keep iterating; not production software and not a clinical system.

Current prototype

Designed for fast, local ward-task capture

  • Free-text-first job creation with optional structured fields.
  • Gesture-handler radial menus for frequently used note and location shortcuts.
  • Configurable local shortcuts, with the first favourites kept wheel-sized for one-handed use.
  • Status, urgency, job-type tags, pinning, bump-to-top, and duplicate-job actions.
  • Compact card mode and local auto-expiry for dense active lists.
  • No backend, login, cloud sync, analytics, messaging, AI, or EHR integration.

Clinical safety boundary

Not a medical record. Not decision support.

This is a personal prototype for transient task organisation only. It is not a referral app, EHR, messaging system, NHS-approved product, clinical safety system, regulated medical device, or GDPR-assured service. It should not be used as the source of truth for patient care.

Runtime data is intended to remain on the device. No patient-identifiable data should be transmitted to this website or repository.

Clinical Shift Scratchpad is an experimental personal project by Sangeev. Review any generated or copied text carefully before using separate clinical systems.
